# 內容生產指南（給產內容的 agent，逐條遵守）

Schema 真相檔：`src/data/types.ts`（含每欄位的格式、字數、枚舉白名單）。
交付物：自包含的 `src/data/cities/<cityId>.ts`，格式：

```ts
import type { Restaurant } from "../types";

export const restaurants: Restaurant[] = [
  // ...
];
```

交付前必須自跑 `npm run check -- src/data/cities/<cityId>.ts` 到全綠（WARN 要逐條說明理由）。

## 選店標準（寧缺勿濫）

1. **知名度門檻**：廣為人知的老店/名店/米其林必比登等級——Google 搜尋「<城市> 美食」「<城市> 必吃老店」穩定出現在前段班的店。
2. **兩來源交叉**：店名、大致位置、招牌菜至少兩個獨立來源一致才收。查不到第二來源就換一家，**禁止腦補**。
3. **排除歇業風險**：近兩年有歇業/搬遷報導的店不收；不確定是否還在營業就換一家。我們寧可少收 20 家，不收 1 家鬼店。
4. **座標**：小數 4 位以上，必須落在該城 bbox 內（見 `src/data/cities.ts`）。座標取店面實際位置，不是商圈中心。不確定的座標寧可重查。
5. **blurb 自撰**：60–120 字、兩句話——它是什麼＋為什麼值得去。繁中台灣用語。**禁抄任何媒體/部落格/評論網文字**（版權線），禁模板句：「必吃」「CP值高」「銅板美食」「饕客」「老饕」。寫具體的事實（賣什麼、什麼做法、什麼場景），不寫空泛形容詞。
6. **id**：kebab-case、`<cityId>-` 前綴、英文語意化（店名音譯或意譯），例 `chiayi-liu-turkey-rice`。全檔唯一。
7. **菜系分布**：照下方每城配額，避免同質（不要整城都是麵店）。夜市攤商菜系照食物本身分（蚵仔煎=street-food），`area` 填夜市名+tag 加「夜市」。
8. **價位帶**：1=~150 銅板 / 2=150–400 / 3=400–1000 / 4=1000+。小吃攤幾乎都是 1，熱炒海產多是 2–3，宴席/割烹才是 4。

## 每店欄位提醒

- `mustOrder`：招牌必點 1–3 樣，每樣 ≤10 字，是推薦引擎的口味訊號（例：愛牛肉麵的人會被跨城推牛肉麵），寫食物名不寫形容詞。
- `tags`：2–6 個，只能取自 `types.ts` 的 TAGS 白名單。
- `since`：查得到創業年代才填（約略十年可，如 1949），查不到省略，禁猜。
- `gmapsQuery`：「店名 城市」，例 `"劉里長雞肉飯 嘉義"`。不是 URL。
- `tips`：≤60 字實用提示（何時去排隊少/怎麼點/收攤早），可省略。
